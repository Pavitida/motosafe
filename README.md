##System Development Procedure

The MotoSafe project was developed as a web-based prototype for riding safety monitoring. At the beginning, the main idea was to create a system that could use smartphone sensors to collect riding data, detect risky events, and display the results clearly on a website.
The first step was developing the web interface. The structure of the website was created using HTML in the index.html file. This file was used to arrange the main sections of the system, such as the dashboard, control buttons, ride information, charts, map area, and CSV tools. After that, CSS in the style.css file was used to design the appearance of the website so that the layout would be clear, organized, and suitable for real-time use.
Next, JavaScript in the script.js file was used to implement the main functions of the system. This part handled GPS tracking, motion-sensor input, event detection, map visualization, CSV import and export, and risk-zone generation. In this stage, the system was tested locally in the browser to check whether the web page could display ride data correctly and whether the event-detection logic worked as expected.
After the local web prototype was working properly, the project files were uploaded to GitHub. This step was used for storing the code, managing project versions, and organizing the final prototype. After that, the uploaded files could be used as the source of the web project, allowing the system to be presented as a browser-based interface instead of only a local file in the development environment.
Once the web system was stable, more functions were added, including CSV export for saving ride logs and CSV import for loading previous ride data back into the system. The map output was also improved by showing detected events as points and grouping repeated nearby events into broader risk zones.
Finally, after the web development part was completed, the collected dataset was prepared for machine learning analysis. The ride logs were cleaned and organized, and a Random Forest model was used to classify riding events. This made the project more complete because it combined web-based monitoring, map-based visualization, and data analysis in one system.
